Travel Advisor

Book your destination through Travel Advisor! Plenty of popular destinations that you can find here!

About this project

This project was created to facilitate bookings and help people find their dream destinations! We are happy to help and find the peoples favorite destinations!

Important information

To access this API user has to be signed in or logged in!

Access API

To access API correct endpoints should be used!

API Endpoints

http://localhost:8000/homepage

This endpoint consists of GET and POST request. The GET reqeust if fetched gives the user random destinations of images and capital name of an extern API in Json format. The POST request is used to give the user a status in Json format if user is logged in or not with userId.

http://localhost:8000/homepage/signin

This endpoint has one POST request that handles the data when user is creating an account. The endpoint handles the database for all the users and adds an user if signed in. It later returns the new created accounts unique id in json format.

http://localhost:8000/homepage/loggedin

This endpoint also consists of one request, POST there it handles the users password and username. The endpoint check the databse if the correct user is given through input and returns a Json formated status text.

http://localhost:8000/searchpage

This endpoint take an GET request there it gives the user all of the destinations from databse in Json format. Therefore the user can search the desired destination and read more.

http://localhost:8000/informationpage

This endpoint takes an POST there it take the given user input of the destination that is clicked. It returns the Json formated destination image and captial name that is used in a seperate page.

http://localhost:8000/destination/wishlist

This endpoint takes an POST and through the POST request it adds the users desired favorite destination to the database and gives a status in Json format back if success

http://localhost:8000//friends/list

This endpoint takes GET, POST and DELETE as requests. The GET request gives the user the whole databse of users in Json format where friends list can be accessed. The POST request gives the option to add a friend where the databse is uppdated through POST. By using the DELETE request, the user can delete a friend through the database. The user is informed with a Json formated status after request.

http://localhost:8000/favorites

This endpoint takes only a POST request were the user can add favorite destinations to a list on the database. The POST updates the list for the user.

http://localhost:8000/booked/loggedin

This endpoint takes a POST request there it gives the user a new instance of class in Json format so users has a history of all the bookings that has been placed. This is returned with a Json formated status text.

http://localhost:8000/friends/list/user

This endpoint also only takes an POST request and uses that to get the loggedin users friends wishlist by the friends information. The POST returns the users wishlist and username in a Json format.
